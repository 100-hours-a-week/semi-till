## 1️⃣주제 1: FE/BE 분리

### FE,BE를 분리하는 이유

1. **확장성**: 각 서버를 독립적으로 확장할 수 있어, 트래픽 증가에 유연하게 대응할 수 있다
2. **보안**:  백엔드 서버를 외부에 노출하지 않기 때문에 보안성에 좋다.
3. **유지 보수성 :** 코드 베이스가 분리되어 있어, 각 기능을 독립적으로 업데이트하고 유지보수 할 수 있다.
4. 성능 최적화 : 프론트와 백엔드에 서로 다른 기술 스택을 사용할 수 있어, 각 부분에 최적화된 기술을 선택할 수 있다.

### FE,BE 분리 기준

| 구분 | 백엔드 (Backend) | 프론트엔드 (Frontend) |
| --- | --- | --- |
| **역할** | 데이터 처리, 서버 로직, API 제공 | 사용자 인터페이스, 동적 웹 동작, API 호출 |
| **주요 요소** | 서버 코드 (`controllers`, `routes`, `models`) | HTML, CSS, JS 등 클라이언트 코드 |
| **의존성** | Express, 데이터베이스 라이브러리 | 브라우저에서 실행되는 JavaScript, Fetch API |
| **실행 위치** | 서버 (Node.js 환경) | 브라우저 (클라이언트 측) |
